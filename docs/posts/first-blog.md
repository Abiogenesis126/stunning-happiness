# 我的第一篇博客：开源软件开发学习心得

## 前言

这是我的第一篇博客文章，记录了我在开源软件开发课程中的学习收获。

## 为什么选择 MkDocs

在众多静态网站生成工具中，我选择了 MkDocs，原因如下：

1. **简单易用**：配置文件简洁，上手快
2. **Markdown 原生支持**：专注内容写作
3. **GitHub Pages 友好**：可以通过 GitHub Actions 轻松部署

## GitHub Actions 自动部署

通过配置 GitHub Actions 工作流，每次提交代码后，网站会自动构建并发布到 GitHub Pages，无需手动操作。

## 总结

通过这次实践，我掌握了：
- GitHub 仓库的基本使用
- Markdown 写作规范
- GitHub Actions 自动化部署
- GitHub Pages 网站发布
